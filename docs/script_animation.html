Script animation

Introduction

A clip is an instruction for applying a verbal and/or nonverbal behaviour based on an extended Behaviour Markup Language (BML). Each clip has an id, start time and duration. Users have the flexibility to modify these parameters, along with other variable properties. The realization of each behavior can be divided into phases, with each phase marked by a sync-point denoting the associated transition. Notably, these sync-point values are falling within the range [0, duration]. The time and the sync-point parameters are in seconds.

Visualization

The color of a clip depends on its type behaviour. Upon selecting a clip, its highlighter and name color shift to white. The fade-in and fade-out animations of the clip are denoted by intensified colors. The fade-in signifies when the action is achieved, while the fade-out marks the point when the character begins returning to their default state.

Selection of a Hand orientation clip in the timeline.

A panel opens to the designated area on the right, displaying all properties of the selected clip. The panel consistently presents the clip Id, a Content section, and a Time section. Depending of the clip type, there may also be Sync-points section. Various other content properties can vary based on the clip's characteristics, which are explained in the Clip types section.

  • Mouthing

    • Generation of lip movements from a text in ARPABET 1-letter notation. In which, '.' and ' ' symbols are also supported.

    • Id

      Identifier name. The default value is Mouthing.

      • Content

    • Text

      String of the text which is wanted to be simulated in the form of ARPABET 1-letter.

    • Optionals

    • Speed *

      Phonemes per second of the whole string. Humans speak at 8 phonemes per second (lower boundary).

    • Intensity *

      Visual exaggeration of phonemes. In the interval [0,1].


    • * It can be specified individually for each phoneme. Turn ON the Set for each phonema property.

      • Time

    • Start

      Second at which the mouthing begins.

    • Duration

      Duration time, in seconds, of the utterance. Automatically computed using the content properties.

Head behaviours:

  • Head movement

    Head movement retrieved from a gesticon by requesting the associated lexeme.

    • Id

      Identifier name. The default value is the name of the lexeme.

      • Content

    • Lexeme

      Name of the particular head behavior from the available list.

      List of lexemes
      Nod
      Shake
      Tilt
      Tilt left
      Tilt right
      Tilt forward
      Tilt backward
      Forward
      Backward

    • Repetitions

      Number of times the basic head motion is repeated.

    • Intensity

      A float value between 0..1 to indicate the amount of the movement. 0 means immeasurable small; 0.5 means "moderate"; 1 means maximally large.

      • Time

    • Start

      Second at which the preparation phase of the movement starts.

    • Duration

      Duration time, in seconds, of the head motion.

      • Sync-point

    • Ready

      Second at which the preparation phase ends. In the interval [0, duration].

    • Stroke start

      Second at which starts the stroke. In the interval [0, duration].

    • Stroke

      Second at which the head motion strokes. In the interval [0, duration].

    • Stroke end

      Second at which the stroke ends. In the interval [0, duration].

    • Relax

      Second at which the retraction phase starts. In the interval [0, duration].

  • Gaze

    Temporarily directs the gaze of the character towards a target.

    • Id

      Identifier name. The default value is the name of the gaze influence.

      • Content

    • Influence

      Determines what parts of the body to move to effect the gaze direction: Eyes, Head, Neck.

    • Target

      A reference towards a target instance that represents the target direction of the gaze.

      List of targets
      Front
      Left
      Right

      Down left
      Down right

    • Base gaze

      Turning ON permanently change the gaze direction of the character towards the target.

    • Optionals

    • Offset angle

      Adds an angle degrees offset to gaze direction relative to the target in the direction specified in the Offset direction. In degrees. By default is 0.0.

    • Offset direction

      Direction of the Offset angle> angle. Same directions as target property.

      • Time

    • Start

      Second at which the gaze starts to move to new target.

    • Duration

      Duration time, in seconds, of gaze returned to default direction.

      • Sync-point

    • Ready

      Second at which the gaze target is acquired. In the interval [0, duration].

    • Relax

      Second at which the gaze starts returning to default direction. In the interval [0, duration].

    Raises the elbow (added to the elbow raise automatically computed while moving the arm).

      Identifier name. The default value is Elbow Raise.

    • Amplitude

      A float value between 0..1 to indicate the amount of the movement. 0 means immeasurable small; 0.5 means "moderate"; 1 means maximally large.

Editing

Animations can be created in three ways:

  • Add clip: Create the animation from scratch adding a BML clip.
  • Add preset: Create the animation from a predefined set of clips based on common patterns.
  • Add sign: Create the animation from an existing glossa or sentence in the repository.

To select one of the options, you can press the buttons on the right panel or right click directly on the timeline.
